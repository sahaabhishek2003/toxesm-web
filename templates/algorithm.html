Algorithm

ToxESM is a peptide toxicity prediction framework that combines transformer-based protein language models with machine learning classifiers. The system converts peptide sequences into high-dimensional representations and uses trained models to classify them as toxic or non-toxic.

Figure: Overview of the ToxESM pipeline showing sequence processing, ESM-2 embedding generation, machine learning classification, and output analysis.

1. Sequence Input and Preprocessing

Users can provide peptide sequences either by pasting them in FASTA/plain format or by uploading a FASTA file. Each sequence undergoes preprocessing:

  • Conversion to uppercase format
  • Removal of non-standard amino acids
  • Validation of sequence length (2–50 residues)

Sequences that do not meet these criteria are not processed for prediction and are marked accordingly in the output.

2. Feature Extraction using ESM-2

ToxESM uses the ESM-2 (Evolutionary Scale Modeling) protein language model (esm2_t33_650M_UR50D) to generate numerical representations (embeddings) for each peptide sequence.

The model treats amino acids as tokens and applies a transformer-based architecture with self-attention to capture:

  • Sequence context and residue interactions
  • Evolutionary and functional patterns
  • Implicit structural information

For each sequence, residue-level embeddings are generated and then averaged (mean pooling) to produce a fixed-length feature vector representing the entire peptide.

3. Machine Learning-Based Classification

The generated embeddings are used as input features for machine learning classifiers trained on curated peptide datasets.

The model was trained on a dataset of 8,828 peptide sequences, consisting of:

  • 4,414 toxic peptides
  • 4,414 non-toxic peptides

The system was evaluated on an independent test set of 2,208 peptide sequences and further validated on external datasets to ensure generalization performance.

The training and test datasets used in this study are available for download from the Download section of this webserver.

The following models are available:

  • XGBoost – High-performance gradient boosting model
  • LightGBM – Efficient boosting model optimized for speed
  • Extra Trees – Ensemble of randomized decision trees

Each model outputs a probability score between 0 and 1, representing the likelihood of the peptide being toxic. A threshold of 0.5 is used for classification:

  • Score ≥ 0.5 → Toxic
  • Score < 0.5 → Non-Toxic

4. Conservative Ensemble Mode

In addition to individual model predictions, ToxESM provides a Conservative Ensemble Mode designed for safety-critical analysis.

In this mode, predictions from all three models (XGBoost, LightGBM, Extra Trees) are combined using a strict rule:

  • If any model predicts "Toxic" → Final prediction = Toxic
  • Only if all models predict "Non-Toxic" → Final prediction = Non-Toxic

This approach minimizes false negatives and ensures that potentially toxic peptides are not overlooked, making it suitable for high-sensitivity screening applications.

5. Residue-Level Importance Analysis

ToxESM provides a residue-level interpretability module to identify which amino acids contribute most to the predicted toxicity of a peptide.

This analysis uses an occlusion-based approach, where each residue is systematically removed and the resulting change in prediction score is measured.

  • The original toxicity score is computed for the full sequence
  • Each residue is removed one at a time to generate mutated sequences
  • The prediction score is recalculated for each mutated sequence
  • The difference in score quantifies residue importance

Residues that cause a larger decrease in predicted toxicity when removed are considered more important for the model's decision.

To improve interpretability, negative contributions are discarded, small values are treated as noise, and importance scores are normalized across the sequence.

The results are visualized using a color gradient:

  • Grey → No significant contribution
  • Light red → Weak contribution
  • Dark red → Strong contribution

This analysis reflects model behavior and provides insight into residue-level influence, but does not necessarily imply biological causation.

6. Physiochemical Property Calculation

In addition to toxicity prediction, ToxESM computes several physiochemical properties using the ProtParam module:

  • Sequence Length
  • Molecular Weight
  • Isoelectric Point (pI)
  • Net Charge
  • GRAVY Index (hydrophobicity)
  • Instability Index
  • Aromaticity
  • Molar Extinction Coefficient

Users can select which properties to display in the results table.

7. Output Generation

The results are presented in a tabular format including:

  • Input sequence and header
  • Selected physiochemical properties
  • Model prediction(s) and probability score(s)
  • Final classification (for ensemble mode)

Results can also be downloaded as a CSV file for further analysis.

8. Prediction Constraints

Predictions are reliable only for peptides within the defined length range (2–50 amino acids). Sequences outside this range or containing invalid residues are excluded from prediction.

Residue-level analysis is computationally intensive and is therefore limited to single valid sequences within the supported length range.
